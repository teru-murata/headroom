title: Text & Log Compression
description: Specialized compressors for search results, build logs, diffs, and general text. Each preserves what matters for its content type.
---

Headroom provides specialized compressors for text-based content that isn't JSON or source code. Each one understands the structure of its content type and preserves what the LLM needs while dropping the noise.

| Compressor | Input Type | What It Preserves | Typical Savings |
|---|---|---|---|
| `SearchCompressor` | grep/ripgrep output | Relevant matches, file diversity | 80-95% |
| `LogCompressor` | Build/test logs | Errors, stack traces, summaries | 85-95% |
| `DiffCompressor` | Unified diffs | Changed lines, context | 60-80% |
| `FileTreeCompressor` | Repository tree/path listings | Package roots, configs, source/test shape, referenced paths | 50-90% |
| `TextCompressor` | General text | Relevant paragraphs, anchors | 60-80% |
| `KompressCompressor` | General text fallback | Learned token scoring via ONNX | 30-50% |

## SearchCompressor

Compresses search results (grep, ripgrep, ag) while keeping the matches that matter.

```python
from headroom.transforms import SearchCompressor

search_results = """
src/utils.py:42:def process_data(items):
src/utils.py:43:    \"\"\"Process items.\"\"\"
src/models.py:15:class DataProcessor:
src/models.py:89:    def process(self, items):
... hundreds more matches ...
"""

compressor = SearchCompressor()
result = compressor.compress(search_results, context="find process")

print(f"Compressed {result.original_match_count} matches to {result.compressed_match_count}")
print(result.compressed)
```

**What gets preserved:**
- Exact query matches (lines containing the search term)
- High-relevance matches (scored by BM25 similarity)
- File diversity (results from different files are kept)
- First/last matches (context from start and end)

### Configuration

```python
from headroom.transforms import SearchCompressor, SearchCompressorConfig

config = SearchCompressorConfig(
    max_results=50,                # Keep up to 50 matches
    preserve_file_diversity=True,  # Ensure different files represented
    relevance_threshold=0.3,       # Minimum relevance score to keep
)

compressor = SearchCompressor(config)
```

## LogCompressor

Compresses build and test output while preserving errors, warnings, and summaries.

```python
from headroom.transforms import LogCompressor

build_output = """
===== test session starts =====
collected 500 items
tests/test_foo.py::test_1 PASSED
... hundreds of passed tests ...
tests/test_bar.py::test_fail FAILED
AssertionError: expected 5, got 3
===== 1 failed, 499 passed =====
"""

compressor = LogCompressor()
result = compressor.compress(build_output)

print(result.compressed)
print(f"Compression ratio: {result.compression_ratio:.1%}")
```

**What gets preserved:**
- Errors and failures (any line with ERROR, FAILED, Exception)
- Warnings
- Full stack traces for debugging
- Test/build summary lines
- Section headers (structural markers like `=====`)
- Coding-agent failure evidence: original command, exit code, failed test name,
  assertion/build diagnostic, and relevant file/line references

**What gets dropped:**
- Hundreds of `PASSED` lines
- Verbose success output
- Repeated patterns

When CCR is enabled and the log is large enough to emit a retrieve marker, the
omitted original remains retrievable through Headroom's local CCR store while
the entry is available.

### Coding-agent failure evidence

For coding agents, a compressed test/build log is only useful if the next step
is still obvious. `LogCompressor` therefore preserves the concrete failure
evidence needed to continue an implementation task:

| Evidence | Examples |
|---|---|
| Original command | `$ python -m pytest tests/test_checkout.py::test_discount_rounding -q` |
| Exit code | `exit code: 1`, `exit code: 101`, `exit code: 2` |
| Failed test name | `FAILED tests/test_checkout.py::test_discount_rounding`, `--- FAIL: TestRetryBackoff` |
| Assertion/root-cause message | `AssertionError`, `expected delay 200ms, got 20ms`, `error TS2345` |
| File and line reference | `tests/test_checkout.py:42`, `src/cart.test.ts:88:21`, `crates/checkout/src/lib.rs:128:9` |
| Build diagnostic | Maven/Surefire failure, TypeScript compiler error, cargo panic location |

The compressor still collapses passing test noise, repeated dependency fetch
lines, progress chatter, and repeated warnings where possible. It does not
promote every generic `ERROR:` line to must-keep status, because that would
break the configured line budget. The evidence guard targets lines that let an
agent identify the failed command, failed test/build target, location, and first
root-cause diagnostic.

Acceptance fixtures for pytest, Vitest, cargo, Go test, Maven, and noisy npm
build output are checked in under `tests/fixtures/log_crusher/`. The acceptance
tests assert both sides of the contract: critical failure evidence survives, and
repeated passing/download noise is reduced.

See [Acceptance Evidence](/docs/acceptance-evidence) for the fixture matrix and
the checked-in validation logs.

## DiffCompressor

Compresses unified diffs while keeping the actual changes and enough context to understand them.

```python
from headroom.transforms import DiffCompressor

diff_output = """
diff --git a/src/main.py b/src/main.py
--- a/src/main.py
+++ b/src/main.py
@@ -42,7 +42,7 @@
 def process(items):
-    return [x for x in items]
+    return [x.strip() for x in items if x]
"""

compressor = DiffCompressor()
result = compressor.compress(diff_output)
```

Diff acceptance coverage checks that compressed patches retain file paths and
status markers, hunk headers, changed public symbols, nearby import changes,
test-impact paths, and local CCR retrievability for omitted diff content.

## FileTreeCompressor

Compresses repository tree output and path listings while keeping the parts a
coding agent needs to choose edit targets.

```python
from headroom.transforms import FileTreeCompressor

tree_output = """
.
|-- package.json
|-- src/
|   `-- server/
|       `-- auth.ts
|-- tests/
|   `-- server/
|       `-- auth.test.ts
`-- node_modules/
    `-- package_001/
"""

compressor = FileTreeCompressor()
result = compressor.compress(tree_output, context="src/server/auth.ts")
```

**What gets preserved:**
- Package roots and local configuration files
- Source and test directory shape
- Changed or recently referenced paths from the context
- Depth-limited tree structure around relevant areas

**What gets collapsed:**
- `node_modules`, `vendor`, `dist`, `build`, `coverage`, `.cache`, and similar generated/cache directories
- Long repeated generated path lists

## Coding-Agent Preset

`CodingAgentPreset` routes noisy coding-agent context by source type instead of
blindly compressing source code.

```python
from headroom.presets import CodingAgentPreset

preset = CodingAgentPreset()
result = preset.route_and_compress(
    "test_log",
    pytest_output,
    {"command": "python -m pytest tests/test_checkout.py -q"},
)
```

Supported local routes include test logs, build logs, search results, file
trees, git diffs, package metadata, MCP/tool responses, and generic noisy tool
output. These are deterministic local transformations, not LLM summarization.

The preset keeps source categories separate:

| Source category | Preservation contract |
|---|---|
| Source code | Preserved verbatim by default; Headroom does not blindly compress source code. |
| Test/build logs | Preserve command, exit code, failed test/build target, assertion or diagnostic, traceback evidence, and file/line references. |
| Diffs | Preserve file paths, file status, hunk headers, changed symbols, nearby imports, and test-impact hints where detectable. |
| File trees | Preserve package roots, source/test directory shape, config/build files, and changed or recently referenced paths. |
| Search results | Preserve file paths, line numbers, matches, and file diversity. |
| Package metadata | Preserve package name, scripts, dependencies, toolchain, and build hints. |
| MCP/tool outputs | Preserve tool name, status, errors, result counts, and the first relevant result. |

When a preset omits substantial output, it uses the existing local CCR
marker/store flow so the full original can be retrieved locally while the store
entry is available. Hosted/team marker policy, Redis/Valkey profiles, and SaaS
retrieve authorization remain outside Headroom's local preset layer.

## TextCompressor

General-purpose text compression with anchor preservation. Best for documentation, README files, and prose content.

```python
from headroom.transforms import TextCompressor

long_text = """
... thousands of lines of documentation ...
"""

compressor = TextCompressor()
result = compressor.compress(long_text, context="authentication")

print(result.compressed)
```

**What gets preserved:**
- Paragraphs relevant to the context query
- Headers and section markers
- Document structure and organization

## Kompress

```python
from headroom.transforms.kompress_compressor import KompressCompressor

compressor = KompressCompressor()
```

## Content Type Detection

If you're building your own routing logic, you can use the content type detector directly:

```python
from headroom.transforms import detect_content_type, ContentType

content = "src/main.py:42:def process():"

detection = detect_content_type(content)
if detection.content_type == ContentType.SEARCH_RESULTS:
    result = SearchCompressor().compress(content, context="process")
elif detection.content_type == ContentType.BUILD_OUTPUT:
    result = LogCompressor().compress(content)
elif detection.content_type == ContentType.FILE_TREE:
    result = FileTreeCompressor().compress(content, context="src/main.py")
elif detection.content_type == ContentType.PLAIN_TEXT:
    result = TextCompressor().compress(content, context="process")
```

## When Each Compressor Is Used

The ContentRouter selects the right compressor automatically. Here's when each fires:

| Content Pattern | Compressor | Detection Signal |
|---|---|---|
| `file:line:content` lines | SearchCompressor | grep/ripgrep output format |
| pytest, npm, cargo markers | LogCompressor | Build tool output patterns |
| `---/+++` and `@@` markers | DiffCompressor | Unified diff format |
| `tree`, `find`, or path-listing output | FileTreeCompressor | Branch markers and path shapes |
| Prose, documentation | TextCompressor | Fallback for non-structured text |
| Long plain text | KompressCompressor | ContentRouter fallback |

## Performance

| Compressor | Typical Input | Output | Speed |
|---|---|---|---|
| SearchCompressor | 1,000 matches | 30-50 matches | ~2ms |
| LogCompressor | 5,000 lines | 100-200 lines | ~3ms |
| DiffCompressor | Large diff | Changed hunks only | ~2ms |
| FileTreeCompressor | Large repository tree | Structural edit-target tree | ~1ms |
| TextCompressor | 10,000 chars | 2,000 chars | ~2ms |
| KompressCompressor | Plain text | 50-70% of original | model-dependent |
